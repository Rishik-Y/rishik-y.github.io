Google Summer of Code 2025 with Waycrate

Making Wayshot Cross-Compatible Across All Wayland Compositors

Project Overview

This website documents my Google Summer of Code 2025 project, where I worked on making Wayshot - a screenshot tool for Wayland compositors - cross-compatible across all Wayland-based compositors, not just wlroots-based ones.

üéØ Project Goal

Transform Wayshot from a wlroots-only screenshot tool to a universal Wayland screenshot solution by implementing the new standardized protocols: ext-image-capture-source-v1 and ext-image-copy-capture-v1.

‚ùå The Problem

Wayshot currently uses the wlr-screencopy protocol, which is non-standard and only works on wlroots-based compositors like Sway and River. This means it can't be used on compositors like:

  • GNOME (Mutter)
  • KDE (KWin)
  • COSMIC
  • Other non-wlroots compositors

‚úÖ The Solution

Implement support for the new official Wayland protocols that provide:

  • Standardized method for display capture
  • Top-level window capture capabilities
  • Cross-compositor compatibility
  • Future-proof architecture
  • Backward compatibility with legacy systems

üéØ Area Selection

Select and capture custom rectangular regions

üé® Pixel Color Grab

Utility to grab RGB values of individual pixels

üîÑ Automatic Fallback

Seamlessly falls back to legacy protocols when needed

üîî Desktop Notifications

Privacy-aware notifications for successful captures
